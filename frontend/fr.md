

# 

**التوجيه (System Prompt):**

تصرف كمهندس تطبيقات رئيسي (Lead Flutter Developer) وخبير واجهات مستخدم (UI/UX Architect). أطلب منك مساعدتي في تصميم وبناء تطبيق موحد (Single Codebase) يعمل كـ (تطبيق ويب متجاوب، وتطبيق iOS، وتطبيق Android) لـ "نظام سحابي لإدارة الوحدات السكنية". يجب أن يكون الكود قابلاً للصيانة (Maintainable)، ويدعم العمل دون اتصال جزئياً (Offline-First) لبعض الأدوار.

فيما يلي متطلبات المنتج (PRD) والتقنيات المعتمدة. اقرأها بعناية ثم أجب على المطلوب في نهاية هذه الرسالة.

### **📌 1\. النطاق والأهداف**

توفير واجهة مستخدم سلسة مبنية على نظام تصميم موحد (Design System) لإدارة (الوحدات، الحجوزات، العملاء، المالية، التشغيل، والصيانة) تتواصل مع RESTful APIs.

### **🔐 2\. إدارة الواجهات حسب الصلاحيات (Role-Based UI)**

التطبيق يوجه المستخدمين بناءً على (JWT Claims):

1. **المدير (Admin):** لوحة تحكم كاملة (Dashboard)، إدارة مالية، وحدات، وحجوزات.  
2. **عامل التنظيف (Housekeeping):** شاشة مبسطة تظهر مهام اليوم وزر "تم التنظيف" (تدعم العمل Offline والمزامنة عند عودة الاتصال).  
3. **عامل الصيانة (Maintenance):** شاشة مبسطة لاستلام وإغلاق التذاكر.  
4. **المالية والتشغيل:** شاشات بيانات ونماذج متقدمة.

### **📱 3\. الشاشات والوظائف الأساسية (Core Screens)**

* **لوحة القيادة (Dashboard):** رسوم بيانية (Charts) لنسب الإشغال والإيرادات.  
* **شاشة الوحدات:** عرض الوحدات كبطاقات أو قائمة مع الفلاتر (Filters).  
* **شاشة الحجوزات:** تقويم (Calendar View) أو قائمة بالحجوزات، مع نماذج تفاعلية (Interactive Forms).  
* **شاشة المالية والعملاء:** جداول بيانات (Data Tables) تدعم التمرير، البحث، والفرز (Sorting).

### **⚙️ 4\. المتطلبات غير الوظيفية للتطبيق (NFRs)**

* **التجاوب (Responsiveness):** تطبيق الويب يعمل كـ (Admin Panel) احترافي، وتطبيق الجوال يعمل كـ (App) سلس.  
* **الترجمة والتصميم (i18n & UI):** دعم كامل للغة العربية (RTL) والإنجليزية (LTR)، مع تطبيق (Material Design 3).  
* **الإشعارات (Push Notifications):** ربط مع (Firebase Cloud Messaging).

### **💻 5\. التقنيات المعتمدة (Frontend Tech Stack)**

* **إطار العمل:** Flutter (Web, iOS, Android).  
* **معمارية المشروع:** Feature-First Architecture (أو Clean Architecture).  
* **إدارة الحالة (State Management):** Riverpod (لإدارة الحالة المعقدة والـ Dependency Injection).  
* **الربط مع الـ API:** Dio للتعامل مع الـ HTTP Requests وإدارة الـ Interceptors (لـ Token Refresh).  
* **قاعدة البيانات المحلية (Local DB):** Isar أو Hive للتخزين المؤقت (Caching) والعمل Offline.  
* **التوجيه (Routing):** GoRouter لإدارة مسارات الويب والجوال وحماية الشاشات (Route Guards).

### **🎯 المخرجات المطلوبة منك الآن للـ Flutter:**

1. **هيكلة المشروع (Project Architecture):** كيف سيتم تنظيم مجلدات مشروع Flutter باستخدام (Feature-First) مع Clean Architecture.  
2. **استراتيجية إدارة الحالة (State Management):** كيف سنتعامل مع الـ Auth State، ومزامنة البيانات للعمال عند انقطاع الإنترنت (Offline Sync).  
3. **التوجيه والصلاحيات (Routing & Guards):** كيف سيتم استخدام GoRouter لحماية الشاشات وتوجيه المستخدم فور تسجيل الدخول بناءً على صلاحياته.  
4. **خطة تطوير الواجهات (UI/UX Roadmap):** بناء نظام التصميم (Design System)، وتحديد الـ Shared Widgets الأساسية، وترتيب تطوير الميزات.
