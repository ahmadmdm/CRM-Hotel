# **البرومبت الأول: مخصص للواجهة الخلفية (Backend & Database)**

**التوجيه (System Prompt):**

تصرف كمهندس برمجيات رئيسي (Lead Backend Engineer) ومهندس معماري لقواعد البيانات (Database Architect). أطلب منك مساعدتي في تصميم وتطوير بنية الواجهة الخلفية لـ "نظام سحابي لإدارة الوحدات السكنية (الإيجار القصير والطويل)". يجب أن يُبنى النظام كـ (Enterprise-Grade Application) يعتمد على بنية (API-First) لخدمة تطبيقات ويب وجوال، مع التركيز العالي على الأداء، الأمان، وقابلية التوسع.

فيما يلي متطلبات المنتج (PRD) والتقنيات المعتمدة. اقرأها بعناية ثم أجب على المطلوب في نهاية هذه الرسالة.

### **📌 1\. النطاق والأهداف**

بناء نظام مركزي (RESTful APIs) لإدارة الوحدات، الحجوزات، العملاء، المالية، التشغيل، والصيانة مع أتمتة دورة حياة الوحدة السكنية، ودعم المهام المجدولة (Background Jobs).

### **🔐 2\. الأمان وإدارة الصلاحيات (Security, RBAC & Auth)**

* النظام محمي بـ JWT مع تطبيق أفضل ممارسات الأمان (OWASP Top 10)، وتحديد معدل الطلبات (Rate Limiting).  
* الأدوار:  
  1. Super Admin: وصول كامل.  
  2. Sub-Admin: وصول تشغيلي، بدون تعديل/حذف للبيانات المالية التاريخية.  
  3. Financial: المدفوعات، المصروفات، التقارير.  
  4. Operations: الحجوزات، حالة الوحدات.  
  5. Maintenance: إدارة تذاكر الصيانة.  
  6. Housekeeping: تغيير حالة مهام التنظيف.

### **🏢 3\. الوحدات الأساسية للداتا (Data Entities & Domain Models)**

* **الوحدات (Units):** الاسم، الرمز، الصور، السعر، الموقع، المرافق، كود القفل الذكي، والحالة (شاغرة، محجوزة، مشغولة، بانتظار تنظيف، جاهزة، تحت الصيانة).  
* **الحجوزات (Bookings):** ربط العميل بالوحدة، (Check-in/out)، التكلفة، الضرائب، مبلغ التأمين، حالة الدفع، وقناة الحجز.  
* **العمليات (Operations):** مهام التنظيف وتذاكر الصيانة (مفتوح، جاري، منتهي، عاجل).  
* **المالية (Finance):** سجلات الإيرادات والمصروفات المربوطة بالوحدات.  
* **العملاء (CRM):** بيانات العميل، سجل الحجوزات، وحالة الحظر (Blacklist).

### **🔄 4\. دورة الحياة المؤتمتة (State Machine Logic)**

الـ API يجب أن يدير حالة الوحدة (Transitions) تلقائياً: إنشاء حجز (محجوزة) ← دخول العميل (مشغولة) ← خروج (بانتظار تنظيف) ← تم التنظيف (جاهزة).

### **💻 5\. التقنيات المعتمدة (Backend Tech Stack)**

* **الإطار واللغة:** Python مع FastAPI (لاستغلال الـ Async/Await و Pydantic).  
* **معمارية الكود:** Clean Architecture أو Domain-Driven Design (DDD).  
* **قاعدة البيانات:** PostgreSQL مع SQLAlchemy (أو SQLModel) وإدارة الهجرات بـ Alembic.  
* **التخزين:** AWS S3 (لرفع الصور والإيصالات).  
* **المهام المجدولة:** Celery مع Redis (للإشعارات وتغيير الحالات).  
* **الاختبار (Testing):** Pytest لعمل (Unit & Integration Tests).  
* **النشر:** Docker و Docker Compose.

### **🎯 المخرجات المطلوبة منك الآن للـ Backend:**

1. **تصميم قاعدة البيانات (Database Schema & ERD):** اكتب لي الجداول الأساسية والعلاقات بينها والـ Indexes المقترحة لتحسين الأداء.  
2. **هيكلة المشروع (Folder Structure):** كيف سيتم تنظيم مجلدات مشروع FastAPI بناءً على (Clean Architecture).  
3. **تصميم واجهات برمجة التطبيقات (API Endpoints):** قائمة مبدئية بأهم مسارات الـ API مع مراعاة (Versioning /api/v1، و Pagination، و Standardized Error Responses).  
4. **استراتيجية المهام المجدولة (Background Tasks):** كيف سنستخدم Celery/Redis برمجياً لإدارة تغير حالات الحجوزات.
